Cloudflare Pages is one of the best deployment platforms available today. Edge-native, globally distributed, and tightly integrated with Cloudflare's ecosystem — it is the right choice for Astro projects that need both static performance and server-side functionality.

This post walks through the complete setup: from a fresh Astro project to a live, production-ready deployment on Cloudflare Pages.

## Why Cloudflare Pages

Before the setup, it's worth being clear about why Cloudflare over Vercel or Netlify.

**Edge-first by default.** Your site runs on Cloudflare's global network — 300+ locations worldwide. Static assets and server functions run close to your users, not in a single region.

**Workers for server-side code.** API routes in Astro become Cloudflare Workers automatically. No cold starts, no region selection, no separate function deployment.

**Free tier that actually matters.** 500 builds per month, unlimited requests, unlimited bandwidth on the free plan. For most projects, this is everything you need.

## Step 1: Install the Cloudflare adapter

```bash
pnpm add @astrojs/cloudflare
```

## Step 2: Configure `astro.config.mjs`

```javascript
import { defineConfig } from 'astro/config';
import cloudflare from '@astrojs/cloudflare';

export default defineConfig({
  output: 'static',
  adapter: cloudflare({
    imageService: 'passthrough',
  }),
});
```

Set `output: 'static'` — this is the correct mode for Astro 6. Pages prerender to static HTML by default. Any API route that exports `export const prerender = false` runs as a Worker.

## Step 3: Configure `wrangler.toml`

```toml
name = "your-project"
compatibility_date = "2026-01-31"
compatibility_flags = ["nodejs_compat"]

pages_build_output_dir = "./dist"
```

The `nodejs_compat` flag is critical. Without it, Node.js APIs used by packages like `resend` will fail at runtime on Cloudflare's edge.

## Step 4: API routes

Any `.ts` file in `src/pages/api/` must explicitly opt out of prerendering:

```typescript
export const prerender = false;

import type { APIRoute } from 'astro';

export const POST: APIRoute = async ({ request }) => {
  // runs on Cloudflare Workers
};
```

## Step 5: Environment variables

Set environment variables in the Cloudflare Pages dashboard under **Settings → Variables & Secrets**. Use **Text** for non-sensitive values, **Secret** for API keys.

Variables only take effect on new deployments — trigger a redeploy after saving.

## Step 6: Connect your repository

In the Cloudflare Pages dashboard, connect your GitHub repository. Every push to `main` triggers an automatic build and deployment. No additional CI configuration required.

## Common issues

**`ERR_PNPM_OUTDATED_LOCKFILE`** — Your `pnpm-lock.yaml` doesn't match `package.json`. Run `pnpm install` locally and commit the updated lockfile.

**`Could not resolve X imported by Y`** — A package has a peer dependency that isn't installed. Add it explicitly: `pnpm add @react-email/render`.

**`output: "hybrid" has been removed`** — Astro 6 merged `hybrid` into `static`. Change your output to `'static'`.

## The result

A Cloudflare Pages deployment gives you a globally distributed site with automatic CI/CD, edge-native server functions, and a free tier that covers most production workloads. The setup takes under an hour. The result is infrastructure that scales to any traffic level without intervention.
